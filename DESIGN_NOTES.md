# 0.3C 视觉说明

## 视觉方向

0.3C 的目标是让「吗喽的出走」先像一个干净、好读、白底为主的小程序，再在第二眼里看到精致的花纸细节。信息层级永远优先于装饰。

花纸风格不是整屏密集底纹，而是白/米底 + 精致局部装饰。

## 花纸参考图样总结

参考图样共同呈现出剪纸、窗棂、卷草纹、花卉纹、云纹、边框纹和护身符式章牌感。它们的重点不是写实插画，而是对称、克制、细线条和局部装饰。

## 页面背景原则

- 大面积使用白色、奶油白、暖米白。
- 背景只保留极轻纸张感。
- 不使用整页厚重花纹背景。
- 装饰只出现在边角、标题区、卡片边框、章节分隔和底部花边。

## 纹样使用位置原则

- 页面标题区：小印章、细花条、角花。
- 重点卡片：护身符式边框和角花。
- 普通卡片：细边框、浅金棕内线。
- 分组标题：ribbon-title 花条。
- 空状态和提示：保持干净，只用轻虚线边框。

## 色彩系统

基础底色：奶油白、米白、暖白。
文字：深棕、灰棕、深灰。
装饰点缀：靛蓝、石榴橙、豆沙粉、深紫、孔雀绿、少量正红、浅金棕边线。

每个页面尽量只有一个主装饰色，避免平均铺满粉绿黄。

## 卡片系统

- paper-card：基础纸张卡片。
- ornament-card：带花纸边饰的卡片。
- amulet-card：护身符式重点卡片。
- sticker-card：轻贴纸小卡。
- note-card：便签纸条。
- archive-card：档案纸条。
- empty-card：空状态卡。
- ribbon-title：分组标题条。
- tiny-seal：小印章装饰。
- corner-flower：角花装饰。
- floral-divider：花条分隔。

## 按钮系统

主按钮像护身符标签，次级按钮像细边纸签，危险按钮使用深紫红或深棕边线，不使用刺眼大红。小按钮像小印章或小签条。

## 后续角色图接入原则

后续角色图应作为内容主体的补充，而不是压过文本和功能。角色图应预留在标题区、状态卡或模块入口旁的小面积位置，不破坏白底留白和花纸边饰秩序。

## 本轮明确不做

- 不新增功能。
- 不改动数据结构。
- 不引入外部图片或网络资源。
- 不加入角色头像或写实插画。
- 不做复杂动画。
- 不做整屏密集花纹背景。


## 0.3C.1 可见性修复

0.3C.1 将关键花纸装饰从“主要依赖伪元素和很淡的线条”调整为“真实 WXML 节点 + 更清晰的装饰色”。首页、泡泡机、日记、仪表盘、出走计划、心声和工具页都加入了肉眼可见的角花、花纸分隔线、护身符边框、地图路线或纸条边饰。

本次仍坚持：先是干净白底和清楚内容，再在标题、边框、角落和分隔处看到精致花纸细节。

## MVP 0.4 视觉识别度打磨

MVP 0.4 的重点是主要页面视觉识别度打磨和试用前体验收口，不新增功能，不改动核心数据结构，不接入登录、云开发、AI、角色图片或外部网络素材。

本轮视觉原则是：第一眼能看到花纸细节，第二眼仍然觉得页面干净、克制、可读。整体继续保持白底 / 米底小程序气质，接近白底花纸手账、护身符和小王国地图，而不是国潮海报、春节祝福页、满屏底纹、普通粉绿模板或商务蓝白风。

本轮主要落点：

- 首页强化「吗喽王国地图」标题牌、地点入口、小印章、路标和路线感。
- 写日记页强化为「护身符记录纸」，分区卡片像一张张记录纸，选项按钮像小标签，分区之间有可见花纸分隔条。
- 日记列表强化为一组被保存下来的纸条，卡片增加纸张感、角花和信息层级。
- 日记详情强化完整事实纪要为「可复制的证据纸」，复制按钮为印章式按钮。
- 决心仪表盘强化为「Croissant 状态护身符」，保护建议像一张小签文或护身符纸条。

当前版本仍不包含登录、云开发、AI、真实社区、医疗诊断、法律咨询。日记、愿望清单、测试内容和本地心声仍然保存在当前设备本地存储中。

## MVP 0.5 今日职场猩象

「今日职场猩象」是世界观入口，不是真登录页，也不是账号、授权、云端身份或用户系统。它放在首页首屏，用一张今日职场能量卡把用户带入吗喽王国。

视觉关键词：星象卡、花纸边框、护身符、白底 / 米底、细线、角花、小印章。卡片可以有星点、细线圆环和花纸分隔，但仍然保持大留白，不使用外部图片、网络素材、角色头像或满屏底纹。

内容原则：荒谬但清醒，可爱但不轻浮，像运势卡但不是算命。文案应该帮助用户观察职场消耗、整理事实和补现实安全感，而不是制造命运感或催促重大决定。

展示原则：每个指标使用「指标名 + 状态值 + 一句短解释」，避免只给百分比或让用户看不懂指标含义。五个指标固定为耗电量、边界天气、废话云层、顺毛概率、安全感库存。

安全原则：不鼓励冲动离职，不做医疗 / 法律 / 命运判断，不使用命令式重大决策、诊断式结论或命运判断式表达。推荐行动优先引导记录具体事件、拆解模糊评价、照顾身体和检查现实准备项。

## MVP 0.6 底部主导航

0.6 的信息架构原则：首页不再承载所有功能入口。首页负责今日状态和今日行动，底部 tabBar 负责主模块切换。

五个主入口职责：

- 今日：今日职场猩象、Croissant 状态简卡、今日推荐行动和高频入口。
- 日记：日记本、离职判断记录、日记详情、事实纪要、删除日记。
- 出走：离职前准备、离职后 30 天重建、我想找回的生活和计划事项完成。
- 百宝箱：泡泡机、深呼吸、自测、急救卡片、出走小技能、隐私说明和试用反馈。
- 心声：本地匿名树洞、精选心声墙和用户自己的本地心声。

视觉原则：tabBar 是结构，不是装饰主角。页面仍保持白底 / 米底、花纸细节、护身符感和小王国气质，原生 tabBar 保持干净、低干扰。

避免事项：不要把首页重新堆成全功能长列表；不要为了装饰做复杂自定义 tabBar；不要引入外部图标素材；不要让底部导航抢走页面本身的花纸手账气质。

## MVP 0.7 角色系统原则

角色是陪伴者，不是催促者。角色文案应该像朋友递来一句话，帮助用户整理事实、照顾身体和补现实安全感，不制造打卡压力或任务焦虑。

角色是轻量提示，不是页面主角。角色图应住进白底 / 米底花纸卡片里，头像适合小提示，半身图适合首页、保存页和呼吸页，不做大海报、不满屏贴纸、不削弱页面主要任务。

四个角色的使用边界：

- Croissant：状态和自我陪伴，适合首页、仪表盘和被接住的承接时刻。
- Elodie：清醒和事实整理，适合写日记、日记详情、事实纪要和安全边界提示。
- Gapchick：休息和恢复，适合深呼吸、情绪急救、旧版顺毛记录承接和出走后的生活重建。
- LD：职场废话来源，只在泡泡机里低频乱入，不进入首页、呼吸、急救或用户承接页面。

安全原则：不鼓励冲动离职，不做医疗 / 法律 / 命运判断，不使用角色制造焦虑、羞耻或任务压力。角色可以提醒“识别即可，不必内化”“不急着做决定”“先让身体知道现在是安全的”。

## MVP 0.7.1 角色素材使用原则

基础头像 / 半身用于默认陪伴，状态图 / 动作图用于更具体的页面语境。图片变化服务于情绪和功能理解，不是为了装饰堆叠，也不应该让页面变成角色展示页。

Croissant 状态图与现有磨损状态绑定：毛很顺、有点炸毛、被搓成毛球、进入保护模式分别使用对应半身图。该绑定只影响展示，不改变磨损值计算或状态分段。

Elodie 的提醒、鼓励、思考要区分使用：提醒用于开始记录前，鼓励用于完成记录后的承接，思考用于事实纪要和线索整理。

Gapchick 的休息、呼吸、递饮料要对应恢复场景：休息用于出走重建，呼吸用于深呼吸练习，递饮料用于急救和正面承接。

LD 的废话、指责、被拆穿素材只作为备用资产保留。0.8D 后泡泡机不再以 LD 为主，LD 只低频乱入，不进入用户承接、呼吸或首页陪伴场景。

透明背景素材应像贴纸一样融入白底 / 米底花纸卡片。图片本身不加白色硬底，不裁切主体，不拉伸，不制造白底矩形感；卡片仍保留花纸边框、角花、小印章和护身符感。

## MVP 0.7.2 角色素材验收与拥挤感微调

本轮确认全部角色素材齐备：Croissant、Elodie、Gapchick、LD 的基础头像 / 半身图，以及 13 张状态 / 动作半身图都已在本地 `assets/characters` 下存在。没有素材缺失，也没有使用假图、外部网络图片或临时替代资源。

角色展示继续遵守“住进卡片里的朋友”原则。角色卡图片使用 `aspectFit`，图片本身保持透明背景，不加白色硬底；半身图尺寸和卡片内距略微收紧，LD 半身图保持更小、更克制，避免压过正文和核心功能。

页面层级继续以内容优先：首页第一视觉仍是今日职场猩象，Croissant 状态只做陪伴和状态简卡；写日记页由 Elodie 轻声提醒；日记详情里的 Elodie 像整理事实和线索；深呼吸、急救和出走重建由 Gapchick 承接；泡泡机由 Gapchick 和 Elodie 共同承接，LD 只低频乱入。

拥挤感调整原则保持不变：增加留白优先于增加装饰，每页只保留关键陪伴，不为了角色系统而塞图。白 / 米底、局部角花和花纸边框继续服务于阅读，而不是成为页面主角。

边界继续保持：不新增登录、授权、云开发、AI、后端、真实社区、外部网络图片、支付、医疗诊断、法律咨询或劳动仲裁建议；不改动本地存储 key 和既有数据结构。

## MVP 0.7.3 试用反馈与隐私说明原则

试用反馈只作为本地纸条盒，不是客服系统、工单系统或真实社区入口。反馈类型、文本内容和创建时间仅保存在当前设备本地存储中，key 为 `malo_trial_feedback_posts`；没有上传、登录、授权、身份关联或图片附件。

反馈入口放在百宝箱底部和隐私说明页中。首页不新增入口，避免干扰今日职场猩象。反馈页可使用 Elodie 做轻量整理陪伴，但角色卡保持小尺寸，不做角色海报。

隐私说明要清醒、温柔、明确：当前试用版不会上传数据；日记、愿望清单、测试内容、树洞内容和试用反馈都在本机；清空缓存、换设备或删除数据后可能无法恢复。不要承诺绝对安全，也不要写成冗长法律条款。

安全边界继续明确：产品不是医疗诊断、心理治疗、法律咨询、劳动仲裁建议或离职决策工具。如果用户正在经历严重身体不适、持续惊恐、强烈自伤念头或无法保证安全，应优先联系现实中的可信任的人、当地紧急帮助或专业支持。

## MVP 0.8A 导航与首页结构原则

本轮优先增强识别，而不是增加功能。底部 tabBar 保持五个主入口：今日、日记、出走、百宝箱、心声。当前 assets 目录下只有角色素材，没有统一风格的小尺寸 tabBar 图标；角色半身图不适合塞进 tabBar，LD 也不应作为主入口图标，因此本轮不接入图标、不生成新素材、不使用网络图片。

后续 tabBar 图标素材建议补齐为一组统一线条 / 花纸小物：今日使用护身符卡、今日能量卡或小花签；日记使用小本子、折角纸或花纸记录纸；出走使用小行囊、路牌或小地图旗帜；百宝箱使用小盒子或护理工具袋；心声使用小纸条、对话框或树洞。

今日页的结构原则是“只负责今天”。今日职场猩象是第一视觉重点；Croissant 只保留一张今日状态小卡，不再重复出现陪伴卡和状态卡；快捷入口只保留记录、急救、计划三个轻量行动，不再承担全站入口职责。

日记页的仪表盘入口只做摘要，不复制完整仪表盘。用户进入日记页后能先看到 Croissant 最近状态和查看仪表盘按钮，但日记页仍以记录列表和新增记录为主，不制造打卡压力。

## MVP 0.8B 泡泡机与填写栏原则

0.8B 曾把泡泡机推进到“荒谬废话标本 + 安静地拆穿它”的方向。这个方向在 0.8D 被确认不符合产品原意，已纠偏为朋友泡泡为主、LD 低频乱入。

泡泡机继续保持白底花纸、半透明泡泡、低饱和细线和克制留白；不生成新素材，不使用网络图片，也不做黑红压迫风或骂战页面。

结果卡像一张拆解说明纸：先说明“已识别职场废话，建议不要内化”，再把废话归类为模糊评价、责任转移、KPI 压迫、情绪操控或空泛鼓励。行动按钮保持轻量，不鼓励冲动离职。

填写栏统一原则：长提示不要塞进 placeholder，改为字段上下方 helper text；placeholder 只保留短句。input / textarea 需要足够 min-height、padding 和 line-height，保证小屏下提示和用户输入都不被裁切。

## MVP 0.8C 日记主流程与文案原则

本轮将日记主流程从正面 / 负面记录收敛为「离职判断与归因复盘」。新建记录不再让用户选择正面或负面，前台语义围绕影响离职判断的事件、反复消耗的因素、边界被碰到的时刻、想离开的理由、想靠近的生活线索和现实下一步。

为了兼容既有 Croissant 磨损值计算，新记录底层继续保存为 `type: "negative"`，同时增加 `entryKind: "decision_factor"` 作为轻量兼容字段。旧版 `positive` 数据不迁移、不删除，列表和详情中显示为「旧版顺毛记录」，仍按旧规则减少磨损值。

事实纪要改为证据式结构：事件事实、影响程度、主要归因、身体 / 情绪反应、已尝试处理、离职判断线索、想靠近的方向和下一步。它只帮助用户整理线索，不提供医疗、法律、劳动仲裁或离职决策建议。

文案池调整方向是真诚、清醒、有回信感：先接住，再提醒，再给一个很小的下一步。保留一点荒谬幽默，但不轻浮；不使用「你值得」「一切都会好起来」这类泛化安慰，也不模仿任何具体在世作家的标志性写法。

安全边界继续保留：当前试用版不上传数据，日记、愿望清单、测试内容、树洞内容和试用反馈都保存在当前设备本地；清空缓存、换设备或删除数据后内容可能无法恢复；当前版本不做登录、授权、云开发、后端数据库、AI 或真实社区；产品不是医疗诊断、心理治疗、法律咨询、劳动仲裁建议或离职决策指令工具。

## MVP 0.8D 泡泡机纠偏原则

泡泡机的正确定位是“两个朋友从不同角度安慰用户的小装置”，不是 LD 废话展示器、职场废话识别器或戳破工具。页面默认必须先看到泡泡机主素材和两条朋友泡泡。

Gapchick 负责安抚型朋友泡泡：放慢、喘气、身体安全感、允许休息，不催促用户振作。Elodie 负责整理型朋友泡泡：清醒、事实、边界、线索整理，不替用户做决定。LD 只作为低频噪音偶尔乱入，概率约 18%，视觉和文案都弱于朋友泡泡。

泡泡机主素材使用用户提供的 `assets/bubble/bubble-machine.png`，通过 `image mode="aspectFit"` 接入，不拉伸、不裁切、不添加白色硬底，也不使用外部网络图片。泡泡效果由 WXML / WXSS 绘制，不新增泡泡图片素材。

操作区以「再吹一次」为主按钮，「记进日记」跳转 `pages/diary-new/index`，「我先收下这两句」只做轻量 toast。页面不制造打卡压力，不鼓励冲动离职，不提供医疗、法律、劳动仲裁或离职决策建议。

## MVP 0.9A 泡泡机点击交互原则

泡泡机在 0.9A 中进一步收束为“朋友吹来一句话”的小装置。页面初始状态保留一片舒适留白，不自动展示泡泡；用户点击靠下的泡泡机素材后，才在上方舞台区出现一个朋友泡泡。

本轮泡泡只来自 Elodie 和 Gapchick：Elodie 使用浅绿 / 奶油绿泡泡，偏事实整理和边界提醒；Gapchick 使用浅粉 / 奶油粉泡泡，偏休息、身体和情绪承接。头像使用 `assets/characters/elodie-avatar.png` 与 `assets/characters/gapchick-avatar.png`，保持小尺寸 `aspectFit`，不变成大角色图。

页面上一次只显示一个泡泡。再次点击泡泡机时，旧泡泡先用轻量 scale + opacity 动画破掉 / 消失，再生成新泡泡。主素材使用 `assets/bubble/bubble-machine.png`，不拉伸、不裁切、不加白色硬底，也不新增图片素材。

0.9A 暂不显示 LD 泡泡，不再保留“再吹一次 / 记进日记 / 我先收下这句”按钮区，避免把泡泡机带回废话识别器、职场吐槽墙或决策建议机。当前仍不接入登录、云开发、AI、后端、真实社区、外部网络图片或支付；产品不是医疗、法律、劳动仲裁或离职决策指令工具。

## MVP 0.9B 泡泡机比例与 tabBar 图标原则

泡泡机页面在 0.9B 中只做比例修正：放大机器本体，移除后方明显粉色椭圆遮罩 / 底托，让下方视觉重心更像真实小装置。泡泡仍在机器上方舞台区出现，初始状态只显示轻提示；页面一次只保留一个朋友泡泡。

tabBar 图标使用 `assets/tabbar-source/` 中的用户提供花纸源素材作为参考和裁切来源，而不是直接把 4096px 大图接入配置。本轮生成的实际图标放在 `assets/tabbar/`，统一为 81px PNG，normal 状态低饱和，active 状态使用对应暖橙、深绿、靛蓝、深紫和浅粉花纸印章。

五个 tab 的映射为：今日使用 warm-orange，日记使用 dark-green，出走使用 indigo-blue，百宝箱使用 deep-purple，心声使用 light-pink；normal 图标统一由 black-white 生成低调灰棕小印章。red-gold 作为强视觉备用源素材保留，不接入底栏。

0.9B 不新增大功能，不恢复 LD 泡泡，不恢复“再吹一次 / 记进日记 / 我先收下这句”按钮，也不接入登录、云开发、AI、后端、真实社区、外部网络图片或支付。
